    • where do we put the style?
      1. can put it directly in the tag (< li style = "colour: purple;") - old, this is not a good idea because we want to keep things separate
      2. style tag: in the head, < style type="text/css"> css here < /style>
        • what happens when there is conflicting style information? (see about-me.html) the one that comes after "wins" - it will override the first instance
        • good for quick, debugging purposes
      3. best: use a link tag - separate .css file
        • use < link rel="stylesheet" href="/path/to/css"> in the head to link to css file
        • in our example, we linked app.css to about-me.html via the href attribute

  • Lecture 48
    • CSS selectors:
    • rememeber that a selector is: selector { ... }
    • element selector:
      • select all instances of a given element
      • ex: select all li's, all div's, etc
    • id selector:
      • add an id as an attribute to any element (as a "hook")
      • will select only the element with that id
      • you need to identify the id in a css with # in front of it
      • #special { ... }
      • great for picking out a single element
    • class selector
      • like an id but you can apply to as many elements on the page as you want
      • you need to identify the class in a css with . in front of it
      • .completed { ... }
      • note: you can make a checkbox already checked when loaded by adding the attribute "checked"
      • < input type="checkbox" name="" value="" checked >
