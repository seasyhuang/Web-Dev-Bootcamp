    • where do we put the style?
      1. can put it directly in the tag (< li style = "colour: purple;") - old, this is not a good idea because we want to keep things separate

        • use < link rel="stylesheet" href="/path/to/css"> in the head to link to css file
        • in our example, we linked app.css to about-me.html via the href attribute

  • Lecture 48: CSS selectors
    • rememeber that a selector is: selector { ... }
    • element selector:
      • select all instances of a given element
      • ex: select all li's, all div's, etc
    • id selector:
      • add an id as an attribute to any element (as a "hook")
      • will select only the element with that id
      • you need to identify the id in a css with # in front of it
      • #special { ... }
      • great for picking out a single element
    • class selector
      • like an id but you can apply to as many elements on the page as you want
      • you need to identify the class in a css with . in front of it
      • .completed { ... }
      • note: you can make a checkbox already checked when loaded by adding the attribute "checked"
      • < input type="checkbox" name="" value="" checked >

  • Lecture 50: More advanced selectors
    • there are at least 30 good ones that you "need to memorize"
    • go over: 5 of those ^
      1. * - selects everything
      2. Descendant selector
        • li a
        • will select everything that is an a inside an li
      3. Adjacent selector
        • h4 + ul
        • siblings: all ul's that come after an h4 (on the same level)
      4. Attribute selector
        • element[attribute]
        • a[href="http://www.google.com"] - will affect all links that go to google
        • input[type="checkbox"] - will select all checkboxes
      5. nth-of-type selector
        • element:nth-of-type(3)
        • li:nth-of-type(3): this will selected the 3rd
        • li:nth-of-type(odd): will select all the odd ones
  • Lecture 52: specificity and the cascade
    • inheritance and specificity
    • li will inherit attributes from its parent: ul or ol
    • any attribute in the parent will affect its children
    • ex: body attributes - affect all children (everything in body)
    • in inspector tool - striked out means the attribute was overrided
    • class will override plain element selector
    • id will override class and plain element selector
    • specificity calculator
