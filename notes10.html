    Server Side Frameworks

  • Lecture 264: introduction to express
    • frameworks: code that other people wrote that we can use
    • framework vs library: library - you're in control; framework - it calls you
    • we want to make things faster, so we use a framework
    • decisions have been made for us, and we fill in the blanks
    • express is a web dev framework (there are many out there)
      • by using express, we can focus on the application and not the setup
      • we use express because it's the most important node web dev framework, there's a lot of tutorials, and it's a lightweight framework
      • heavy vs light weight framework: heavier weight frameworks have fewer blanks to fill in
      • ex: rails is a heavy weight framework - hard to explain, and you have less of idea of what's going on
      • express homepage

  • Lecture 267: the package.json
    • what is the package.json file?
      • javascript object notation
      • all the information about express
      • dependencies - all the packages express relies on
      • imagine you have a great recipe and your friend wants to recreate it - you could send all the ingredients in a big box and send it, or you could just send your friend a letter with all the ingredients
      • the package.json is the shopping list
      • it contains a list of metadata about a package, the version, etc
    • what does the --save mean at the end of npm install express --save ?
      • when we install a package with npm install, we add --save
      • it will take package name and version and automatically save it into our package.json file
    • the line npm init will create a package.json file for us in a new directory (the folder where you want the app to exist)
    • example: npm install cat-me --save will automatically add "cat-me" to our dependencies section of our package.json

    • * route matcher
      • using app.get("*", ... - every other route that isn't defined (good for error message)

    • writing routes with route parameters
      • example: reddit.com
      • routes - the subreddit for /r/soccer
