  • Lecture 32: Intro to Forms
    • use the form tag
    • < form class="" action="index.html" method="post">< /form>
      • action: what url to send form data to (ideally: to our own backend)
      • method: what type of http request (get, post, etc)
      • Searches = get request
      • Sign ups = post request (add to db)
      • adding the form tag makes the form BLOCK
      • otherwise, collection of inputs is just inline
    • input tag is what goes into the forms - user inputs, text, etc
      • < input type="text" name="" value="">
      • type - ex. text, password
      • example in first-page.html
      • name - sent in the http request (pay attention to the url)
      • name attribute is used behind the scenes to understand how to send data
    • Lecture 35: Labels
      • important for visually imparied (screenreaders)
      • put the input inside the label tag. two types of syntax:
        1. < label>Username: < input type="text" value="username">< /label>
        2. < label for="username">Username: < input id="username" value="username2">
        3. link label to input using label for="something" and atrribute ex. radio id
    • Lecture 36: Form validations
      • simple validations: making sure that things are not left blank
      • add the attribute like this: < input required >
      • if you set the type = email, it will make sure that there is an @ sign in the email
    • Lecture 37: Dropdown and Radio Buttons
      • radio - one choice: ex. cats, dogs (must be in the form)
        • < input type="radio" name="something" value="DOG" >
        • set "name" attribute (in 2+ radio inputs) to link the radio buttons
        • if a button is the last thing in the form, will automatically submit
        • value is the thing you're sending
      • select tag - Dropdown menus
        • < option > for choices
        • to change what value is sent (instead of sending text inside the option tag), specify the value attribute
        • set "name" attribute of select to change what is sent in the request (url)
      • checkbox - can choose multiple or one
      • textarea
        • when you want user to be able to submit more text
        • < textarea name="name" rows="8" cols="80">
